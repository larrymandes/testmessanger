## ПОТОК ДАННЫХ (ОБЯЗАТЕЛЬНО СЛЕДУЙ)

```
Событие (IDLE, кнопка, lifecycle)
    ↓
UI вызывает метод сервиса
    ↓
Сервис обрабатывает (fetch, decrypt, save в БД)
    ↓
Сервис уведомляет UI через callback
    ↓
UI загружает из БД
    ↓
UI обновляет отображение (setState)
```

## СЕРВИСЫ - ДЕТАЛИ

### ChatService (главный координатор)
- Инициализирует все сервисы
- Связывает EmailService ↔ MessageService
- Единая точка входа для UI
- Методы: `initialize()`, `fetchAndProcessNewMessages()`, `sendMessage()`, `addUICallback()`, `removeUICallback()`

### MessageService (обработчик сообщений)
- Обрабатывает входящие письма
- Расшифровывает
- Сохраняет в БД
- Уведомляет UI через callbacks
- Методы: `processNewMessages()`, `addUICallback()`, `removeUICallback()`

### EmailService (транспорт)
- IMAP подключение и IDLE loop
- Fetch новых писем
- SMTP отправка
- Вызывает processor для обработки
- Методы: `connectImap()`, `fetchNewMessages()`, `sendMessage()`, `setMessageProcessor()`

## ПРИМЕРЫ ПРАВИЛЬНОГО КОДА

### ✅ ПРАВИЛЬНО - UI вызывает сервис:
```dart
// В ChatListScreen
void _onRefresh() {
  _chatService.fetchAndProcessNewMessages(); // ✅
}
```

### ✅ ПРАВИЛЬНО - Сервис обрабатывает:
```dart
// В ChatService
Future<void> fetchAndProcessNewMessages() async {
  final messages = await _emailService.fetchNewMessages();
  await _messageService.processNewMessages(messages);
}
```

### ✅ ПРАВИЛЬНО - UI только отображает:
```dart
// В ChatListScreen
Future<void> _loadContacts() async {
  final contacts = await StorageService.getContacts(email);
  setState(() { _contacts = contacts; });
}
```

### ❌ НЕПРАВИЛЬНО - Обработка в UI:
```dart
// В ChatListScreen
Future<void> _processMessage(msg) async {
  final decrypted = await CryptoService.decrypt(...); // ❌ НЕТ!
  await StorageService.saveMessage(...);              // ❌ НЕТ!
}
```

## КОГДА ДОБАВЛЯЕШЬ НОВУЮ ФУНКЦИЮ

1. **Спроси себя**: Это бизнес-логика или UI?
   - Бизнес-логика (обработка данных, работа с сервером/БД) → Сервис
   - UI (отображение, анимации, навигация) → Экран

2. **Если бизнес-логика**:
   - Добавь метод в соответствующий сервис
   - UI только вызывает этот метод

3. **Если UI**:
   - Добавь в экран
   - Но если нужны данные - вызови сервис

## ВАЖНЫЕ МОМЕНТЫ

### Fetch при открытии приложения:
- При `AppLifecycleState.resumed` вызывается `ChatService.fetchAndProcessNewMessages()`
- При первом запуске тоже вызывается
- Это гарантирует что сообщения придут даже если приложение было закрыто

### Callbacks:
- UI регистрирует callbacks через `ChatService.addUICallback()`
- Callbacks хранятся в `MessageService._uiCallbacks`
- `MessageService._notifyUI()` вызывает все callbacks после обработки
- UI в callback только загружает из БД и обновляет отображение

### BCC копии:
- Свои BCC копии пропускаются в `MessageService._processMessage()`
- Проверка: `if (from == accountEmail) { skip }`

### Read receipts:
- Отправляются при открытии `ChatScreen`
- Батчинг: все UID одним сообщением
- Обрабатываются в `MessageService._handleReadReceipt()`
