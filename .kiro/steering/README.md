# Steering Rules - Правила для AI

Эти файлы автоматически загружаются AI ассистентом при каждом запросе.

## Файлы:

### `project-rules.md` - Главные правила
- ⚠️ Правила работы AI (читай файлы полностью, проверяй документацию, думай)
- 🏗️ Архитектура проекта
- ✅ Что писать где
- ❌ Что НЕ делать
- 📝 Примеры правильного/неправильного кода

### `code-patterns.md` - Паттерны кода
- 📋 Шаблоны для типичных задач
- 🔧 Как добавлять новые функции
- 🔄 Как обрабатывать lifecycle
- 📊 Как логировать
- ⚠️ Типичные ошибки

### `faq.md` - Частые вопросы
- ❓ Где добавить функцию?
- 🔄 Как обновить UI?
- 🐛 Как отлаживать?
- 📚 Почему нельзя обработку в UI?
- И другие частые вопросы

---

## Для чего это нужно?

✅ AI всегда помнит правила проекта
✅ AI следует архитектуре
✅ AI использует правильные паттерны
✅ Код всегда качественный и правильный

---

## Как это работает?

1. Пользователь задаёт вопрос
2. AI автоматически загружает эти файлы
3. AI следует правилам при ответе
4. Результат: правильный код, правильная архитектура

---

## Важно!

Эти файлы для AI, не для разработчика.

Для разработчика есть:
- `ПРАВИЛА_РАЗРАБОТКИ.md` - правила разработки
- `АРХИТЕКТУРА.md` - описание архитектуры
