# 🔒 Функции безопасности

## Добавлено в production-ready версию

### ✅ 1. Валидация публичных ключей

**Где:** `CryptoService.isValidPublicKey()`

**Проверки:**
- Длина ключа = 130 символов (65 байт в hex)
- Начинается с "04" (uncompressed ECDH key)
- Только hex символы (0-9, a-f)
- Валидная точка на эллиптической кривой secp256r1

**Применяется:**
- При сканировании QR-кода (перед отправкой invite)
- При получении invite (перед сохранением контакта)
- При отправке сообщения (перед шифрованием)

**Защита от:**
- Битых/повреждённых ключей
- Случайных ошибок при сканировании QR
- Злонамеренных действий (фейковые ключи)
- Крашей при шифровании

---

### ✅ 2. Лимит длины сообщений (4096 символов)

**Где:** `ChatService.sendTextMessage()`

**Как работает:**
- Лимит: 4096 символов (как в Telegram)
- Длинные сообщения автоматически разделяются на части
- Части отправляются последовательно
- **Пользователь НЕ видит разделение** - просто пишет как обычно

**Пример:**
```
Сообщение 10000 символов → 3 части:
- Часть 1: 4096 символов (отправляется)
- Часть 2: 4096 символов (отправляется)
- Часть 3: 1808 символов (отправляется)

Получатель видит 3 отдельных сообщения
```

**Защита от:**
- Перегрузки email серверов
- Проблем с шифрованием больших данных
- Потери данных при передаче

---

### ✅ 3. Rate Limiting (2 сообщения в секунду)

**Где:** `ChatService._checkRateLimit()`

**Как работает:**
- Максимум 2 сообщения в секунду
- Автоматическое ожидание при превышении лимита
- **Сообщение показывается как "отправляется"** пока ждём
- Прозрачно для пользователя

**Пример:**
```
Пользователь быстро отправляет 5 сообщений:
- Сообщение 1: отправлено сразу (0ms)
- Сообщение 2: отправлено сразу (0ms)
- Сообщение 3: показывается "отправляется", ждёт 500ms → отправлено
- Сообщение 4: показывается "отправляется", ждёт 500ms → отправлено
- Сообщение 5: показывается "отправляется", ждёт 500ms → отправлено
```

**Защита от:**
- Спама (случайного или злонамеренного)
- Блокировки email сервером
- Перегрузки получателя
