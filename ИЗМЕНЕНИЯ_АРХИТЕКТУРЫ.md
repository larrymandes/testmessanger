```dart
// ChatListScreen - только UI
class _ChatListScreenState extends State<ChatListScreen> {
  late ChatService _chatService;
  
  Future<void> _initialize() async {
    // Инициализация ChatService (он сам делает всё)
    await _chatService.initialize();
    
    // Регистрация callback для обновления UI

      if (mounted) {

    // Загружаем контакты из БД
    await _loadContacts();

  Future<void> _loadContacts() async {
    // Просто загружаем из БД - никакой обработки!
```

## Разделение ответственности

### ChatService (координатор)
- Инициализация всех сервисов
- Связывание EmailService ↔ MessageService
- Управление callbacks
- Единая точка входа для UI

### MessageService (обработчик)
- Обработка входящих писем
- Расшифровка
- Сохранение в БД
- Уведомление UI

### EmailService (транспорт)
- IMAP/SMTP подключение
- Fetch писем
- Отправка писем
- IDLE loop

### UI Screens (отображение)
- Загрузка из БД
- Отображение данных
- Обработка пользовательского ввода
- Вызов методов ChatService

## Преимущества

### 1. Чистота кода
- Каждый компонент отвечает за свою задачу
- Нет дублирования логики
- Понятная структура

### 2. Надёжность
- Единая точка обработки сообщений
- Нет race conditions между экранами
- Все экраны работают одинаково

### 3. Производительность
- Обработка происходит один раз
- UI только обновляется из БД
- Нет повторной обработки

### 4. Масштабируемость
- Легко добавлять новые экраны
- Легко добавлять новые типы сообщений
- Легко тестировать

### 5. Поддерживаемость
- Понятно где что находится
- Легко найти и исправить баги
- Легко добавлять функции

## Решённые проблемы

### ❌ Было: Сообщения не появлялись сразу
**Причина**: ChatListScreen и ChatScreen обрабатывали сообщения параллельно, ChatScreen открывался до того как ChatListScreen закончил обработку.

**Решение**: Теперь обработка происходит один раз в MessageService, а UI экраны просто загружают из БД.

### ❌ Было: Дублирование логики
**Причина**: ChatListScreen и ChatScreen имели одинаковую логику обработки сообщений.

**Решение**: Вся логика в MessageService, UI экраны только отображают.

### ❌ Было: Сложно добавлять новые экраны
**Причина**: Нужно было копировать всю логику обработки.

**Решение**: Новые экраны просто используют ChatService и загружают из БД.

## Как использовать

### Создание нового экрана

```dart
class MyNewScreen extends StatefulWidget {
  final ChatService chatService;
  
  const MyNewScreen({required this.chatService});
  
  @override
  State<MyNewScreen> createState() => _MyNewScreenState();
}

class _MyNewScreenState extends State<MyNewScreen> {
  @override
  void initState() {
    super.initState();
    
    // Регистрируем callback
    widget.chatService.addUICallback(_onNewMessage);
    
    // Загружаем данные
    _loadData();
  }
  
  void _onNewMessage() {
    if (mounted) {
      _loadData(); // Просто перезагружаем из БД
    }
  }
  
  Future<void> _loadData() async {
    // Загружаем из StorageService
    final data = await StorageService.getSomething();
    setState(() { /* обновляем UI */ });
  }
  
  @override
  void dispose() {
    widget.chatService.removeUICallback(_onNewMessage);
    super.dispose();
  }
}
```

## Миграция существующего кода

Если у вас есть старый код, вот как его мигрировать:

### 1. Замените EmailService на ChatService

```dart
// Было
late EmailService _emailService;
_emailService = EmailService(email: email, password: password);

// Стало
late ChatService _chatService;
_chatService = ChatService(email: email, password: password);
```

### 2. Уберите обработку сообщений из UI

```dart
// Было
Future<void> _processMessage(msg) async {
  // Расшифровка, парсинг, сохранение
}

// Стало
// Удалите этот метод - обработка в MessageService
```

### 3. Замените fetch на загрузку из БД

```dart
// Было
await _fetchNewMessages(); // Fetch + обработка

// Стало
await _loadContacts(); // Просто загрузка из БД
```

### 4. Используйте callbacks для обновления

```dart
// Было
_emailService.setNewMessageCallback(() {
  _fetchNewMessages();
});

// Стало
_chatService.addUICallback(() {
  if (mounted) {
    _loadContacts();
  }
});
```
