# Анализ: Наш код vs Delta Chat

## ✅ Что уже сделано правильно (как Delta Chat)

### 1. UIDNEXT вместо UNSEEN ✅
- Используем `selectResult.uidNext` для отслеживания новых писем
- Fetch только новых: `UID FETCH lastUidNext:*`
- Не зависит от флага \Seen
- Быстрее и надёжнее чем SEARCH UNSEEN

### 2. UIDVALIDITY проверка ✅
- Проверяем `selectResult.uidValidity`
- Если изменился → ящик пересоздан, сбрасываем всё
- Защита от проблем с переназначением UID

### 3. Message-ID дедупликация ✅
- Генерируем уникальный Message-ID при отправке
- Сохраняем в БД (`processed_message_ids`)
- Проверяем при получении → пропускаем дубликаты
- Дополнительная проверка `from == myEmail`

### 4. IDLE цикл ✅
- 29 минут (RFC рекомендует до 30)
- DONE → NOOP для проверки соединения
- Переподключение при ошибках
- Не останавливается для fetch/mark

### 5. UID дедупликация ✅
- Таблица `processed_uids`
- Проверка перед обработкой
- Защита от повторной обработки

## ⚠️ Что нужно исправить

### 1. RFC822 → BODY.PEEK[] ✅ ИСПРАВЛЕНО
**Проблема:**
```dart
'(UID RFC822 BODY.PEEK[HEADER.FIELDS (FROM SUBJECT)])'
```
- `RFC822` автоматически помечает письма как \Seen
- Мы теряем контроль над флагом
- Письма помечаются ДО обработки

**Решение:**
```dart
'(UID BODY.PEEK[] BODY.PEEK[HEADER.FIELDS (FROM SUBJECT MESSAGE-ID)])'
```
- `BODY.PEEK[]` НЕ помечает как \Seen
- Мы сами контролируем когда пометить
- Добавили MESSAGE-ID в заголовки

### 2. Батчинг fetch (опционально)
Delta Chat использует батчи по ~1000 символов в команде (RFC 2683).
У нас сейчас: `UID FETCH lastUidNext:*` - может быть много писем.

**Можно улучшить:**
```dart
// Если новых писем > 100, fetch батчами
if (currentUidNext - _lastUidNext > 100) {
  // Fetch по 100 писем за раз
}
```

Но для мессенджера это не критично - обычно приходит 1-5 писем.

### 3. Обработка своих писем
**Сейчас:**
- Проверяем `from == myEmail`
- Проверяем Message-ID в `processed_message_ids`

**Delta Chat:**
- НЕ загружает свои письма с IMAP вообще
- Использует BCC to self (опция)
- Перемещает в отдельную папку DeltaChat

**Наш подход проще и работает:**
- Свои письма попадают в INBOX
- Мы их фильтруем по Message-ID
- Не нужна отдельная папка
