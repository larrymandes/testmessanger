# 📱 ИНСТРУКЦИЯ ПО ОБНОВЛЕНИЮ

## 🔥 ВАЖНО! Критическое обновление БД

Это обновление изменяет структуру базы данных для правильной синхронизации между устройствами.

---

## ✅ ЧТО ИЗМЕНИЛОСЬ?

### Было:
- Сообщения имели разные ID на разных устройствах
- Read receipts не всегда работали
- Возможны дубликаты сообщений

### Стало:
- Сообщения имеют одинаковый ID на всех устройствах
- Read receipts работают везде
- Дубликаты невозможны

---

## 🚀 КАК ОБНОВИТЬ?

### Вариант 1: Автоматическая миграция (рекомендуется)

1. Обновите приложение на ВСЕХ устройствах
2. Запустите приложение
3. БД автоматически мигрирует
4. Готово!

**Что сохранится:**
- ✅ Все сообщения с Message-ID (новые)
- ❌ Старые сообщения без Message-ID (удалятся)

### Вариант 2: Чистая установка

1. Удалите приложение
2. Установите новую версию
3. Добавьте аккаунт заново
4. Добавьте контакты заново

**Что сохранится:**
- ❌ Ничего (чистый старт)

## 📊 ТЕХНИЧЕСКИЕ ДЕТАЛИ

### Что изменилось в БД?

**Версия 1 → Версия 2:**
- PRIMARY KEY: `id` → `message_id`
- Удалены поля: `id`, `uid`
- Автоматическая дедупликация

### Что такое Message-ID?

Message-ID - это уникальный идентификатор email сообщения (RFC 5322).

**Пример:** `<1234567890.123@mail.ru>`

**Преимущества:**
- Одинаковый на всех устройствах
- Не меняется никогда
- Стандарт для email систем
