# Правила разработки - ОБЯЗАТЕЛЬНО К ПРОЧТЕНИЮ

## 🎯 Главное правило

**ВСЯ БИЗНЕС-ЛОГИКА В СЕРВИСАХ, НЕ В UI!**

---

## 📁 Структура проекта

```
lib/
├── services/           # ВСЯ бизнес-логика ЗДЕСЬ
│   ├── chat_service.dart       # Главный координатор
│   ├── message_service.dart    # Обработка сообщений
│   ├── account_service.dart    # Управление ключами
│   ├── email_service.dart      # IMAP/SMTP транспорт
│   ├── crypto_service.dart     # Шифрование
│   └── storage_service.dart    # База данных
│
└── screens/            # ТОЛЬКО UI
    ├── chat_list_screen.dart   # Список чатов
    ├── chat_screen.dart        # Экран чата
    └── qr_screen.dart          # QR коды
```

---

## ✅ ЧТО ДЕЛАТЬ

### В СЕРВИСАХ (services/):
- ✅ Обработка сообщений
- ✅ Расшифровка/шифрование
- ✅ Fetch с сервера
- ✅ Сохранение в БД
- ✅ Бизнес-логика
- ✅ Валидация данных

### В UI (screens/):
- ✅ Отображение данных
- ✅ Обработка нажатий
- ✅ Навигация
- ✅ Показ уведомлений
- ✅ Копирование в буфер
- ✅ Вызов методов сервисов

---

## ❌ ЧТО НЕ ДЕЛАТЬ

### В UI (screens/):
- ❌ Обработка сообщений
- ❌ Расшифровка
- ❌ Fetch с сервера
- ❌ Прямая работа с БД (только через сервисы)
- ❌ Бизнес-логика

### В СЕРВИСАХ (services/):
- ❌ Работа с UI (setState, Navigator, etc)
- ❌ Показ диалогов
- ❌ Работа с контекстом

---

## 🔄 Правильный поток данных

```
Событие (IDLE, кнопка, lifecycle)
    ↓
UI вызывает метод сервиса
    ↓
Сервис обрабатывает (fetch, decrypt, save)
    ↓
Сервис уведомляет UI через callback
    ↓
UI загружает из БД
    ↓
UI обновляет отображение (setState)
```

## 📝 Примеры ПРАВИЛЬНОГО кода

### ✅ ПРАВИЛЬНО - UI вызывает сервис:

```dart
// В ChatListScreen
void _onRefresh() {
  _chatService.fetchAndProcessNewMessages(); // ✅ Вызов сервиса
}
```

### ✅ ПРАВИЛЬНО - Сервис обрабатывает:

```dart
// В ChatService
Future<void> fetchAndProcessNewMessages() async {
  final messages = await _emailService.fetchNewMessages(); // ✅ Fetch
  await _messageService.processNewMessages(messages);      // ✅ Обработка
}
```

### ✅ ПРАВИЛЬНО - UI только отображает:

```dart
// В ChatListScreen
Future<void> _loadContacts() async {
  final contacts = await StorageService.getContacts(email); // ✅ Загрузка из БД
  setState(() { _contacts = contacts; });                   // ✅ Обновление UI
}
```

## 📝 Примеры НЕПРАВИЛЬНОГО кода

### ❌ НЕПРАВИЛЬНО - Обработка в UI:

```dart
// В ChatListScreen
Future<void> _processMessage(MimeMessage msg) async {
  final decrypted = await CryptoService.decrypt(...); // ❌ Бизнес-логика в UI!
  await StorageService.saveMessage(...);              // ❌ Обработка в UI!
}
```

### ❌ НЕПРАВИЛЬНО - UI в сервисе:

```dart
// В MessageService
void processMessage() {
  setState(() { ... }); // ❌ UI в сервисе!
  Navigator.push(...);  // ❌ Навигация в сервисе!
}
```

## 🎯 Как добавить новую функцию

### 1. Определи где логика:

**Бизнес-логика?** → Добавь в сервис
- Обработка данных
- Работа с сервером
- Работа с БД
- Валидация

**UI логика?** → Добавь в экран
- Отображение
- Анимации
- Навигация
- Уведомления

### 2. Создай метод в сервисе:

```dart
// В ChatService
Future<void> myNewFeature() async {
  // Вся логика здесь
}
```

### 3. Вызови из UI:

```dart
// В Screen
void _onButtonPressed() {
  _chatService.myNewFeature(); // Просто вызов
}
```

## 🔧 Отладка

### Если что-то не работает:

1. **Проверь логи** (LogsScreen)
   - Все логи имеют префикс сервиса
   - Легко найти где проблема

2. **Проверь поток данных**:
   - Событие → Сервис → БД → UI
   - Где оборвалось?

3. **Проверь callbacks**:
   - Зарегистрирован ли callback?
   - Вызывается ли `_notifyUI()`?

## 📚 Архитектура

### ChatService (главный)
- Инициализация всех сервисов
- Связывание EmailService ↔ MessageService
- Единая точка входа для UI

### MessageService (обработчик)
- Обработка входящих писем
- Расшифровка
- Сохранение в БД
- Уведомление UI

### EmailService (транспорт)
- IMAP подключение
- IDLE loop
- Fetch писем
- SMTP отправка

### UI Screens (отображение)
- Загрузка из БД
- Отображение
- Вызов методов сервисов

- [ ] Вся бизнес-логика в сервисах?
- [ ] UI только вызывает методы сервисов?
- [ ] Нет обработки данных в UI?
- [ ] Нет UI кода в сервисах?
- [ ] Callbacks правильно зарегистрированы?
- [ ] Логи добавлены для отладки?
- [ ] Код компилируется без ошибок?

---

## 💡 Золотые правила

1. **Один источник истины** - БД
2. **Одна точка обработки** - MessageService
3. **UI только отображает** - загружает из БД
4. **Сервисы не знают про UI** - только callbacks
5. **UI не знает про детали** - только вызывает методы
